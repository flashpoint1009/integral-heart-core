## الهدف
بناء نسخة احترافية من النظام تدعم:
1. **حسابات جاهزة لكل الوظائف** (8 أدوار)
2. **نظام Modules**: المدير/المطور يفعّل أو يلغي مكونات كاملة (HR، المالية، المخزون...) — أساس البيع بنسخ مختلفة
3. **Multi-tenant**: المطور يدير عدة شركات/عملاء من لوحة واحدة، كل شركة بياناتها معزولة
4. **داشبورد ذكي + توقع مبيعات بالـ AI + تحليل ربحية**

---

## المرحلة 1: البنية التحتية للـ Multi-tenant (الأهم)

### قاعدة البيانات
- جدول `tenants` (شركات/عملاء النظام): الاسم، اللوجو، التاريخ، الحالة (نشط/متوقف)، خطة الاشتراك
- جدول `tenant_modules`: لكل شركة، أي مكونات مفعلة (`hr`, `finance`, `inventory`, `rep`, `accounting`...)
- إضافة عمود `tenant_id` لكل الجداول الموجودة (customers, products, sales_invoices, employees... كلها)
- تحديث كل سياسات RLS لتعزل البيانات حسب `tenant_id`
- جدول `developer_users`: حسابات المطور (Super Admin) — منفصلة تماماً عن مستخدمي الشركات
- دور جديد `developer` فوق `admin` في الـ enum

### Helper Functions
- `current_tenant_id()` — security definer ترجع tenant_id الخاص بالمستخدم الحالي
- `is_developer()` — تحقق من صلاحية المطور
- `module_enabled(tenant_id, module_name)` — تحقق إن المكون مفعل

## المرحلة 3: نظام الـ Modules وإدارة الصلاحيات

- **شاشة "إدارة المكونات"** (للمطور والمدير): توجل ON/OFF لكل مكون
- **إخفاء ديناميكي** للقوائم والصفحات حسب المكونات المفعلة
- **شاشة "صلاحيات الموظفين"** للمدير: استثناءات خاصة لموظفين معينين (إخفاء/إظهار شاشات)
- جدول `user_screen_permissions` للاستثناءات الفردية

## المرحلة 4: لوحة المطور (Super Admin)

مسار `/developer/*` محمي بدور `developer`:
- **قائمة العملاء (Tenants)**: إضافة/تعديل/إيقاف، مع KPIs لكل عميل
- **إنشاء نسخة جديدة**: wizard ينشئ tenant + admin + مكونات مختارة في كليكة
- **مراقبة الاستخدام**: عدد المستخدمين، الفواتير، التخزين لكل عميل
- **سجلات الدخول والنشاط** عبر العملاء
- **إدارة خطط الاشتراك**

## المرحلة 5: الداشبورد والتقارير الاحترافية

### داشبورد تنفيذي `/dashboard/executive`
- KPIs ديناميكية: مبيعات اليوم/الشهر، نمو، أعلى عملاء، أعلى منتجات
- رسوم Recharts: bar/line/area/donut + مقارنات شهرية وسنوية
- خريطة حرارية للزيارات (Leaflet)
- مؤشر صحة الأعمال (Health Score)

### توقع المبيعات بالـ AI `/dashboard/forecast`
- Server Function تستخدم Lovable AI (google/gemini-2.5-flash)
- تحلل آخر 12 شهر مبيعات وتتنبأ بـ 3 شهور قادمة
- توصيات ذكية: "ركز على منتج X"، "موسم بطيء متوقع"
- رسم Forecast vs Actual

### تحليل الربحية `/dashboard/profitability`
- تصنيف ABC للعملاء والمنتجات
- هامش ربح لكل صنف
- العملاء الخاسرين/المربحين

### تقارير المندوبين المتقدمة `/dashboard/reps`
- معدل التحويل (زيارات → مبيعات)
- متوسط الفاتورة
- خريطة حرارية للزيارات
- ترتيب الأداء (Leaderboard)

## التفاصيل التقنية

- 6-7 migrations منفصلة، كل واحدة لها GRANTs و RLS صحيحة
- كل الـ functions الموجودة هتتحدث لتحترم `tenant_id`
