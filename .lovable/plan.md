# خطة التنفيذ بالترتيب

سأنفذ المراحل واحدة تلو الأخرى، وبعد كل مرحلة أوقف عشان تجرب وتأكد قبل الانتقال للي بعدها.

---

## المرحلة 1: Audit Log (سجل العمليات للمطور)

**الهدف:** المطور يرى كل حركة في النظام (مين عملها، إيه، إمتى) ويقدر يمسحها.

- جدول `audit_log` يحتوي: المستخدم، الجدول، نوع العملية (INSERT/UPDATE/DELETE)، البيانات قبل وبعد، التاريخ، IP.
- Triggers تلقائية على الجداول الحساسة: `sales_invoices`, `expenses`, `payments`, `payroll_runs`, `employees`, `products`, `customers`, `suppliers`, `journal_entries`, `online_orders`.
- شاشة `/audit-log` للمطور فقط (role=`developer`):
  - فلاتر: جدول، مستخدم، تاريخ، نوع العملية.
  - عرض تفصيلي للتغييرات (Diff).
  - زر حذف للسجلات.
- صلاحية الوصول مقصورة على role `developer` فقط.

---

## المرحلة 2: تطبيق المندوب Mobile-First (PWA)

- تحويل المشروع لـ PWA قابل للتثبيت (Manifest + Icons + Theme).
- تحسين شاشات المندوب (`/rep/*`) للموبايل:
  - شاشة Check-in/Out بحجم أزرار أكبر وواجهة مبسطة.
  - قائمة العملاء بكروت كبيرة وبحث سريع.
  - زيارات اليوم في صفحة واحدة بتصميم Card-based.
  - Bottom Navigation بدل السايدبار على الموبايل.
- استخدام `vite-plugin-pwa` (manifest فقط، بدون offline حالياً).
- أيقونة "Install App" تظهر للمناديب.

---

## المرحلة 3: النسخ الاحتياطي

- شاشة `/settings/backup` للأدمن:
  - **تصدير كامل:** زر يصدّر كل بيانات الشركة (Customers, Products, Invoices, Employees... إلخ) كملف JSON واحد.
  - **تصدير CSV لكل جدول** على حدة.
  - **استعادة:** رفع ملف JSON واستعادة البيانات (مع تأكيد).
- النسخ الاحتياطي يتم عبر `createServerFn` مع `supabaseAdmin`.
- جدولة يومية تلقائية عبر `pg_cron` (اختياري — تخزين في bucket خاص).

---

## المرحلة 4: الإشعارات (Notifications)

- جدول `notifications`: المستلم، النوع، العنوان، الرسالة، الرابط، مقروء/غير مقروء.
- أيقونة جرس 🔔 في الـ Header مع عداد غير المقروء.
- Drawer لعرض آخر الإشعارات.
- Triggers تلقائية:
  - **طلب أونلاين جديد** → إشعار للأدمن والكاشير.
  - **طلب إجازة جديد** → إشعار للمدير.
  - **موافقة/رفض إجازة** → إشعار للموظف.
  - **مخزون منخفض** (< reorder_point) → إشعار للأدمن (يومياً عبر cron).
  - **فاتورة متأخرة السداد** → إشعار للأدمن (يومياً).
- Realtime عبر Supabase Subscriptions.

---

## المرحلة 5: التقارير و Dashboards

- شاشة `/reports` بتبويبات:
  - **تقرير المبيعات:** حسب الفترة/العميل/المنتج، تجميعات يومية/شهرية، رسوم بيانية.
  - **تقرير المشتريات:** من الموردين، مردودات.
  - **تقرير الأرباح والخسائر:** P&L كامل من الـ Journal.
  - **تقرير المخزون:** القيمة الحالية، الحركة، الأصناف الراكدة.
  - **تقرير الموظفين:** الحضور، الإجازات، الرواتب.
- تصدير كل تقرير لـ **PDF** (jsPDF) و **Excel** (xlsx).
- رسوم بيانية بـ Recharts (موجود مسبقاً).

---

## المرحلة 6: تعدد اللغات (Arabic + English)

- مكتبة `i18next` + `react-i18next`.
- ملفات ترجمة: `src/locales/ar.json`, `src/locales/en.json`.
- ترجمة كل النصوص الثابتة في الواجهة.
- زر تبديل اللغة في الـ Header.
- تبديل اتجاه الصفحة تلقائياً (RTL/LTR).
- حفظ اختيار المستخدم في `profiles.preferred_language`.
- التواريخ والأرقام تنسّق حسب اللغة.
